0cmplx is a query composition engine. Connect your database, and it becomes an MCP server that AI agents can query in natural language.

## The problem

Every database needs an MCP server. Every MCP server needs tool definitions, parameter validation, query logic, and security testing. Multiply that by every database in your organisation and you have a project, not a feature.

## What 0cmplx does

You provide a schema. 0cmplx does the rest.

The engine parses your tables, columns, and foreign keys into a relationship graph. When a question arrives, the engine composes the correct SQL, joins across relationships, and executes in a sandboxed database. The result comes back as structured data.

```
"Get the balance for the user with phone +2781..."

Engine knows:
  users.phone → users.id
  accounts.user_id → users.id
  accounts.balance

Composes:
  SELECT a.balance FROM accounts a
  JOIN users u ON a.user_id = u.id
  WHERE u.phone = '+2781...'

Returns: { balance: 12450.00 }
```

One tool call. No manual joins. No CRUD boilerplate.

## Confidence-based execution

Not every question maps cleanly to the schema. The engine scores its confidence:

- **High confidence**: known tables, columns, and relationships. Executes immediately.
- **Low confidence**: ambiguous terms or missing columns. Returns what it resolved and what it could not, without executing.

The caller rephrases. The engine never guesses.

## Security testing

Every MCP server is an attack surface. 0cmplx includes OWASP-informed security traps that connect to any MCP server and probe for:

- **Injection**: can a parameter carry a payload the server executes?
- **Data exposure**: does a response include fields the caller should not see?
- **Broken access**: can one user's request access another user's data?
- **Resource exhaustion**: can a call consume unbounded resources?

Run `0cmplx test --target <url>` and see what breaks.

## Who it is for

**Developers building AI agents.** You need data access for your agent. You want a secure MCP server from your schema without building one by hand.

**Developers building MCP servers.** You have a server. You want OWASP traps to probe it before your users do.

**Platform teams.** You need a reliable infrastructure layer for tool execution. 0cmplx handles query composition and sandboxing. Your governance layer handles the rules.

## Try, develop, deploy

**Try** with a sample database. No files, no credentials, no sign-up. Pick banking, ecommerce, or healthcare and start querying.

**Develop** with your schema. Upload a SQL file. Test with sandbox data. Iterate until the tools are right.

**Deploy** with credentials. Same schema, same tools, real data. Production credentials stay out of the loop until you are ready.

## The name

0cmplx. Zero complexity. One command to connect your database to AI agents. The complexity is in the engine, not in your workflow.
