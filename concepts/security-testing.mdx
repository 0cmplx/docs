An MCP server is an API that AI agents call. The caller is not a human clicking buttons. It is a language model choosing parameters based on a conversation. It might be helpful. It might also be manipulated.

0cmplx's security traps simulate these scenarios. They connect to your MCP server, call tools with adversarial inputs, and report what happens.

## What the traps test

<CardGroup cols={2}>
  <Card title="Injection" icon="syringe">
    Can a parameter carry a payload the server executes? SQL fragments, shell commands, prompt text that alters downstream behaviour.
  </Card>
  <Card title="Data exposure" icon="eye">
    Does a response include more data than needed? Internal IDs, tokens, debug info that should not reach the model.
  </Card>
  <Card title="Broken access" icon="lock-open">
    Can a tool be called without auth? Can one user access another's data by changing an ID?
  </Card>
  <Card title="Resource exhaustion" icon="battery-empty">
    Can a call consume unbounded resources? Missing pagination, recursive operations, no rate limits.
  </Card>
</CardGroup>

## How it works

A trap connects to the MCP server, discovers available tools, calls them with crafted inputs, and inspects responses. It does not modify your server.

```bash
0cmplx test --target http://localhost:3100
```

Each tool call is evaluated:

- **Pass**: the server handled the input safely.
- **Warn**: behaviour worth reviewing. A response includes fields that look internal, but might be harmless.
- **Fail**: exploitable behaviour. Immediate attention recommended.

## Works on any MCP server

The traps are not limited to servers generated by 0cmplx. Point them at any MCP server: hand-written, generated by another tool, or running in production. Same traps, same report.

## What traps do not do

Traps find common problems. They do not find all problems. They cannot reason about your business logic or know which fields are sensitive in your context.

Think of them as a first pass. They catch what is easy to miss and tedious to test manually. They do not replace a security review, but they make the review faster.
