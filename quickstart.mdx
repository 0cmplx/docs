## Option 1: Try a sample (fastest)

No files. No credentials. Pick a pre-built database and start exploring.

### From the CLI

```bash
npm install -g @0cmplx/cli
0cmplx try banking
```

This loads a banking database with users, accounts, policies, premiums, and claims. You can query it immediately.

### From the dashboard

Go to [0cmplx.com](https://0cmplx.com) and click **Start testing**. An app is created with demo servers pre-installed. Select a server, pick a tool, and execute.

### Available samples

| Sample | Tables | What it covers |
|---|---|---|
| `banking` | users, accounts, policies, premiums, claims, transactions | Financial data, multi-table joins, user lookups |
| `ecommerce` | users, products, orders, order_items, reviews, categories | Product catalogues, order history, aggregations |
| `healthcare` | patients, doctors, appointments, prescriptions, diagnoses | Patient records, scheduling, medical data |

## Option 2: Bring your schema

Upload your own SQL schema file. The engine parses it and builds tools from your tables.

```bash
0cmplx auth
0cmplx schema upload schema.sql --name mydb
```

## Option 3: Connect your database

When you are ready for production, provide database credentials. Same schema, same tools, real data.

```bash
0cmplx connect --host db.example.com --port 5432 --name mydb
```

Start with a sample or schema file. Connect credentials when you are confident in the setup.
