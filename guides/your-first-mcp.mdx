Before using 0cmplx to generate servers from OpenAPI specs, it helps to build one by hand. This guide walks through creating a minimal MCP server, connecting to it, and understanding what happens at each step.

## What we are building

A small MCP server with two tools:

- `greet`: takes a name, returns a greeting
- `add`: takes two numbers, returns the sum

These are deliberately simple. The point is to see the shape of an MCP server, not to build something useful.

## Set up the project

```bash
mkdir hello-mcp && cd hello-mcp
npm init -y
npm install @modelcontextprotocol/sdk
```

## Write the server

Create `server.js`:

```javascript
import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
import { z } from 'zod';

const server = new McpServer({
  name: 'hello-mcp',
  version: '1.0.0',
});

server.tool('greet', { name: z.string() }, async ({ name }) => ({
  content: [{ type: 'text', text: `Hello, ${name}.` }],
}));

server.tool(
  'add',
  { a: z.number(), b: z.number() },
  async ({ a, b }) => ({
    content: [{ type: 'text', text: `${a} + ${b} = ${a + b}` }],
  })
);

const transport = new StdioServerTransport();
await server.connect(transport);
```

That is the entire server. Two tools, each with a typed input schema and a handler that returns a result.

## Run it

```bash
node server.js
```

The server starts and listens over stdio. An AI agent (or any MCP client) can now connect to it, discover the two tools, and call them.

## What to notice

- **Tool names are identifiers.** The model will see `greet` and `add` when it lists available tools. Short, clear names work best.
- **Input schemas use Zod.** The MCP SDK validates inputs against the schema before your handler runs.
- **Handlers return content blocks.** The response is an array of content items. Text is the most common, but the protocol supports images and other types.
- **Transport is separate from logic.** The server does not know or care whether it is connected over stdio, HTTP, or something else. You choose the transport when you connect.

## Now try generating one

The server above is 20 lines. A real API might have dozens of endpoints. Writing each tool by hand is tedious and error-prone. That is what `0cmplx generate` is for.
