---
title: Configuration
description: Reference for 0cmplx.config.json options.
---

The configuration file controls how 0cmplx generates and tests your MCP server. It is created by `0cmplx init` or `0cmplx generate` and lives in your project root as `0cmplx.config.json`.

## Full schema

```json
{
  "apiBase": "https://api.example.com",
  "transport": "stdio",
  "port": 3100,
  "auth": {
    "type": "bearer",
    "envVar": "API_TOKEN"
  },
  "tools": {
    "listPets": {
      "enabled": true,
      "name": "listPets",
      "description": "Custom description for the AI model",
      "confirm": false
    }
  },
  "generate": {
    "skipDeprecated": true,
    "skipWithoutDescription": true,
    "skipFileUploads": true
  },
  "test": {
    "timeout": 5000,
    "categories": ["injection", "exposure", "access", "resource"]
  }
}
```

## Top-level options

| Field | Type | Default | Description |
|---|---|---|---|
| `apiBase` | string | required | Base URL of the API the generated handlers call |
| `transport` | `"stdio"` or `"http"` | `"stdio"` | How the MCP server communicates with clients |
| `port` | number | `3100` | Port for HTTP transport |

## auth

Authentication configuration for outgoing API requests.

| Field | Type | Description |
|---|---|---|
| `type` | `"bearer"`, `"apiKey"`, `"basic"` | Authentication scheme |
| `envVar` | string | Environment variable containing the credential |
| `header` | string | Custom header name (for `apiKey` type) |

```json
// Bearer token
{ "type": "bearer", "envVar": "API_TOKEN" }

// API key in custom header
{ "type": "apiKey", "envVar": "API_KEY", "header": "X-API-Key" }

// Basic auth (username:password in env var)
{ "type": "basic", "envVar": "API_CREDENTIALS" }
```

## tools

Per-tool overrides. Each key is the tool name (derived from `operationId` or the generated name).

| Field | Type | Default | Description |
|---|---|---|---|
| `enabled` | boolean | `true` | Whether to include this tool |
| `name` | string | from spec | Override the tool name |
| `description` | string | from spec | Override the tool description |
| `confirm` | boolean | `false` | Mark as requiring user confirmation before execution |

Setting `enabled: false` keeps the tool file on disk but excludes it from the server's tool list. This is useful for temporarily disabling tools without deleting their handlers.

## generate

Options that control the generation step.

| Field | Type | Default | Description |
|---|---|---|---|
| `skipDeprecated` | boolean | `true` | Skip endpoints marked as deprecated |
| `skipWithoutDescription` | boolean | `true` | Skip endpoints with no summary or description |
| `skipFileUploads` | boolean | `true` | Skip endpoints that accept multipart/form-data |

## test

Options that control the testing step.

| Field | Type | Default | Description |
|---|---|---|---|
| `timeout` | number | `5000` | Timeout in milliseconds for each trap |
| `categories` | string[] | all | Which trap categories to run |

## Environment variable interpolation

String values that start with `$` are resolved from environment variables at runtime:

```json
{
  "apiBase": "$API_BASE_URL"
}
```

This keeps secrets and environment-specific values out of the configuration file.
