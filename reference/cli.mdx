The 0cmplx CLI is a thin client to the 0cmplx cloud. It authenticates with your account, sends commands to the cloud API, and streams results back to your terminal. Nothing runs locally except reading your spec files.

## Install

```bash
npm install -g @0cmplx/cli
```

## Authentication

### 0cmplx login

Authenticate with your 0cmplx account via GitHub OAuth.

```bash
0cmplx login
```

Opens your browser to complete the GitHub OAuth flow. Stores your API token in `~/.0cmplx/credentials`.

### 0cmplx logout

Clear stored credentials.

```bash
0cmplx logout
```

### 0cmplx whoami

Show the current authenticated user and tier.

```bash
0cmplx whoami
```

## Execute

### 0cmplx execute init

Upload a spec or schema to the cloud and create an execution context.

```bash
0cmplx execute init --spec <path> [options]
0cmplx execute init --schema <path> [options]
```

| Option | Description |
|---|---|
| `--spec` | Path to OpenAPI spec or Postman collection (JSON/YAML) |
| `--schema` | Path to SQL schema file (CREATE TABLE DDL) |
| `--seed` | Path to seed data file (SQL INSERT statements or CSV) |
| `--auth` | Auth configuration for API mode (e.g., `bearer:$KEY`) |

The spec is uploaded to the cloud. The cloud parses it, builds the relationship graph, and returns the available functions or tables.

### 0cmplx execute run

Send a prompt to the cloud. The LLM writes a script, the cloud executes it, and results stream back to your terminal.

```bash
0cmplx execute run --prompt <text> [options]
```

| Option | Default | Description |
|---|---|---|
| `--prompt`, `-p` | required | Natural language question |
| `--format` | `text` | Output format: `text` or `json` |
| `--dry-run` | `false` | Show the generated script without executing |

Output streams in real-time: script generation token by token, bridge calls with timing, final results.

### 0cmplx execute inspect

Show the current execution context.

```bash
0cmplx execute inspect
```

Lists available functions (API mode) or tables (SQL mode), auth configuration, and sandbox settings.

## Test

### 0cmplx test

Run OWASP security traps via the cloud against a target MCP server.

```bash
0cmplx test [options]
```

| Option | Default | Description |
|---|---|---|
| `--target`, `-t` | required | URL of the MCP server to test |
| `--category` | all | Trap category: `injection`, `exposure`, `access`, `resource` |
| `--trap` | all | Run a single named trap (e.g., `injection/sql-fragment`) |
| `--tools` | all | Comma-separated list of tool names to test |
| `--format` | `text` | Output format: `text` or `json` |
| `--output` | stdout | Write results to a file |

### Examples

```bash
# Test a running MCP server
0cmplx test --target http://localhost:3100

# Test only injection traps
0cmplx test -t http://localhost:3100 --category injection

# JSON output for CI
0cmplx test -t http://localhost:3100 --format json --output report.json
```

## Exit codes

| Code | Meaning |
|---|---|
| `0` | Success |
| `1` | One or more test failures |
| `2` | Configuration or auth error |

Warnings do not cause a non-zero exit code. If you want to fail on warnings, check the JSON report.
